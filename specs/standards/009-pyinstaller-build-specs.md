# 009-pyinstaller-build-specs.md - Build Specifications Standard

Standardized specification for building the PDF ADA Processor executable with PyInstaller.

## Overview

The project uses **PyInstaller** (not Node.js `pkg`) to bundle Python dependencies and create a standalone Windows executable (`pdf-ada.exe`). This standard ensures consistent builds across different development environments.

## Build Tool: Why PyInstaller over pkg?

| Feature | pkg (Old) | PyInstaller (Current) |
|---------|-----------|----------------------|
| Runtime Bundled | Node.js v18 (~60 MB) | Python + pip deps (~45-76 MB) |
| Platform Support | Windows only | Cross-platform (.exe, .dmg) |
| Build Command | `pkg server.js --targets ...` | `pyinstaller pdf-ada.spec` |
| Dependencies | npm modules + Node runtime | Python packages via pyproject.toml |

**Migration Date:** 2026-04-21 (commit 5af3a35)

## The PDF ADA PyInstaller Specification (`pdf-ada.spec`)

### Core Configuration

```python
# -*- mode: python ; coding: utf-8 -*-

a = Analysis(
    ['python/src/main.py'],          # Entry point - dynamically resolves paths at runtime
    pathex=[],                       # No custom PYTHONPATH needed
    binaries=[],                      # No binary executables to bundle
    datas=[                           # Bundled data files
        ('adobe_credentials.json.template', '.'),  # User template file
        ('server.js', '.'),                    # Web server for browser access
    ],                              # Plus other Python scripts (pipeline, compliance_checker)
    hiddenimports=[                   # Node.js modules that must be bundled
        'express',     # Web UI framework
        'pdf-lib',     # PDF manipulation library
        'multiprocessing',  # Parallel processing utilities
    ],
    hookspath=[],                     # Custom hooks (handled by PyInstaller default hooks)
    hooksconfig={},                   # Hook configurations (default: optimize=2)
    runtime_hooks=[],                 # Runtime hooks for dynamic behavior
    excludes=['tkinter'],            # Exclude GUI modules not used in CLI
    noarchive=False,                 # Include .pyc files
    optimize=2,                      # MAX optimization level
    debug=False,                     # Production builds (not dev)
)
```

### Executable Configuration

```python
pyz = PYZ(a.pure)

exe = EXE(
    pyz,
    a.scripts,                       # Python scripts generated by PyInstaller
    a.binaries,                      # Bundled binaries (if any)
    a.datas,                         # Data files from datas list above
    [],                              # No additional data dirs
    name='pdf-ada',                  # Executable name in dist/
    debug=False,                     # Production builds
    bootloader_ignore_signals=True,  # Faster startup on Windows
    strip=False,                     # Keep symbols for debugging (optional)
    upx=True,                        # Compress with UPX (~40% size reduction)
    console=True,                    # Console mode output
    disable_windowed_traceback=True, # Better error messages in UI
    argv_emulation=False,            # Don't emulate command line parsing
    target_arch=None,                # x86/x64 auto-detected (Windows only)
)
```

## Pain Points & Resolutions

### 1. PATH Variable Interference ✅ RESOLVED

**Problem:** When running from `dist/`, environment variables can pollute `sys.argv`. For example, if PATH contains `/path/to/server.js`, argparse picks it up as an argument.

**Symptom:** 
```
DEBUG: subcommand=server, actual=server, remaining=['C:\Users\rsbiiw\Projects\pdf-ada\server.js']
KeyError: 'C:\\Users\\rsbiiw\\Projects\\pdf-ada\\server.js'  # Not in COMMANDS dict!
```

**Resolution:** In `python/src/main.py`, filter out file paths before passing to subprocess:
```python
filtered_remaining = []
for arg in remaining:
    if isinstance(arg, str) and ('/' in arg or '\\' in arg):
        continue  # Skip full paths (from environment variables)
    filtered_remaining.append(arg)
cmd_args = list(filtered_remaining) if len(filtered_remaining) > 0 else []
```

### 2. Hardcoded Project Paths ⚠️ PARTIAL RESOLUTION

**Problem:** The dispatch function used an absolute path (`C:\Users\rsbiiw\Projects\pdf-ada`) instead of resolving dynamically from the executable location.

**Resolution (in progress):** Changed to `Path(sys.executable).parent.parent` for better portability. However, this still requires the project structure to match exactly.

### 3. Argument Passing Between Layers ⚠️ KNOWN LIMITATION

**Problem:** When arguments are passed through subprocess calls, dry-run mode detection may lose information. Arguments combined via `list(sub_args) + (list(remaining) if remaining else [])`.

**Current Behavior:**
- `--dry-run` flag is correctly detected in main()
- But when dispatched to Python scripts, flags may be duplicated or lost
- Example: `.exe --dry-run ada-auto` becomes `[python.exe, adobe_auto.py, '--dry-run', 'ada-auto']`

**Impact:** Non-critical but can cause confusion for users expecting clean argument flow.

### 4. Cross-Platform Portability ⚠️ INCOMPLETE

**Problem:** The dispatch function currently uses Windows-specific paths (`C:\Users\rsbiiw\Projects\pdf-ada`). On macOS/Linux, the path resolution may differ.

**Resolution Needed:** Use `Path(sys.executable).resolve()` which should work across platforms by resolving from where the executable runs.

## Build Commands

### Automated (Windows)
```cmd
build-pyinstaller.cmd           # Uses PyInstaller with clean builds and error handling
```

### Manual Build Options

#### Basic build:
```bash
# Windows
pyinstaller pdf-ada.spec --clean

# macOS/Linux  
python -m pyinstaller pdf-ada.spec --clean
```

#### Optimized (no UPX compression):
```bash
pyinstaller pdf-ada.spec --clean --upx-exclude
```

#### Debug build:
```bash
pyinstaller pdf-ada.spec --clean --debug
```

## Runtime Execution from dist/ Folder

When running the executable directly from `dist/`, be aware of these patterns:

### Windows (from project root):
```cmd
.\pdf-ada.bat              # Wrapper script, shows help or runs server
.\pdf-ada.exe              # Direct executable call
  # Default behavior: starts server
  .\pdf-ada.exe --help     # Show CLI options
  .\pdf-ada.exe ada-auto input.pdf -o output.pdf  # Run tool with args
```

### macOS/Linux (from dist folder):
```bash
cd dist
./pdf-ada                  # Universal bundle, uses system Python
./pdf-ada server           # Explicit subcommand
--dry-run                 # Dry run mode
```

**Important:** Arguments are passed directly to the executable. For example:
```bash
# This will dispatch 'server' with any trailing args
python dist/pdf-ada.exe --server some-flag
```

## Hidden Imports (Node.js Libraries)

The following Node.js modules must be bundled because `main.py` imports them via `__file__`:

| Module | Purpose | Why Must Be Bundled? |
|--------|---------|---------------------|
| `express` | Web UI framework | Used by `server.js` (bundled as data file) |
| `pdf-lib` | PDF manipulation | Used by Python scripts via subprocess calls |
| `multiprocessing` | Parallel processing utilities | Used for multi-threaded compliance checks |

PyInstaller automatically discovers these through the hiddenimports list in the spec file.

## Dependencies (pyproject.toml)

Python dependencies should be defined in `python/pyproject.toml`:

```toml
[build-system]
requires = ["setuptools>=61.0", "wheel"]
build-backend = "setuptools.build_meta"

[project]
name = "pdf-ada"
version = "2.0.0"
description = "PDF accessibility compliance processor"
authors = [{ name = "PDF ADA Team", email = "team@pdf-ada.com" }]
requires-python = ">=3.11"

# Production dependencies (bundled by PyInstaller)
dependencies = [
    "pikepdf>=6.10.0",      # PDF processing
    "requests>=2.31.0",      # HTTP requests for Adobe API
]

[project.optional-dependencies]
dev = ["pytest", "mypy"]  # Development tools (not bundled)
```

## Data Files Included by Default

The following files are automatically included in the executable:

- `adobe_credentials.json.template` - User template file
- `server.js` - Web server for browser access  
- Python scripts from `python/` directory
- Any `.pyc` or cached files (if optimize=True)
